# Сервис анонимных обращений

Flask-приложение для анонимной отправки сообщений о буллинге и общения с психологом по токену. После отправки обращения текст классифицируется ML-моделью, сохраняется в SQLite, а психолог может отвечать через админ-панель.

## Что есть в проекте

- `flask_app.py` — Flask-приложение, маршруты страниц и JSON API.
- `models.py` — SQLAlchemy-модели для обращений, сообщений, токенов, тегов, журнала действий и админов.
- `templates/` — HTML-страницы сайта: главная форма, чат, вход, админ-панель и информационные страницы.
- `anti_bullying.db` — SQLite-база данных.
- `lgbm_model_toxic_.pkl` — обученная LightGBM-модель для классификации текста.
- `tfidf_vectorizer.pkl` — TF-IDF-векторизатор для подготовки текста перед классификацией.
- `requirements.txt` — зависимости Python.

## Как запустить

Нужен Python 3.11 или новее.

1. Создать и активировать виртуальное окружение:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

2. Установить зависимости:

```bash
pip install -r requirements.txt
```

3. Запустить приложение:

```bash
python flask_app.py
```

4. Открыть в браузере:

```text
http://localhost:8080
```

## Доступ психолога

При первом запуске приложение создает пользователя, если админов еще нет:

```text
Логин: psychologist
Пароль: psychologist123
```

Страница входа: `http://localhost:8080/login`

## Основные страницы

- `/` — отправка анонимного обращения.
- `/chat/<token>` — чат по токену обращения.
- `/login` — вход психолога.
- `/admin` — админ-панель с обращениями и сообщениями.
- `/about`, `/how-it-works`, `/resources`, `/faq`, `/contact`, `/privacy-policy`, `/terms-of-service` — информационные страницы.

## API

Публичные методы:

- `POST /api/report` — создать обращение.
- `POST /api/chat` — отправить сообщение в чат по токену.
- `POST /api/chat/token` — получить историю чата по токену.

Методы админ-панели:

- `POST /api/admin/login` — войти в админ-панель.
- `POST /api/admin/logout` — выйти.
- `GET /api/admin/tickets` — получить список обращений.
- `PUT /api/admin/tickets/<id>/status` — изменить статус обращения.
- `PUT /api/admin/tickets/<id>/priority` — изменить приоритет обращения.
- `GET /api/admin/tickets/<id>/chat` — получить чат обращения.
- `POST /api/admin/tickets/<id>/chat` — отправить ответ психолога.
- `GET /api/admin/messages` — получить список сообщений.

## Примечания

Для запуска нужны файлы модели `lgbm_model_toxic_.pkl` и векторизатора `tfidf_vectorizer.pkl` в корне проекта. База SQLite создается и обновляется при старте приложения через SQLAlchemy.
